How to properly use @font-face to avoid faux-italic and bold browser styles

See the full blog post for a more detailed explanation.

The Problem

If you just apply a single font to a site without creating any additional styles for the bold and italic fonts, then the browser will automatically create 'faux' bold and italics for you, as seen in this demo. Note the second paragraph, which simulates what happens if the font fails to load, still has appropriate bolds and italics.

Note: I know the difference between the real fonts and the faux styles are subtle. Try comparing this sample to the other solutions. In particular, you can see that the lower-case letter 'a' in the italic sample is different in the true font. Also the faux-bold works by stretching the text horizontally, resulting in wide vertical strokes and thin horizontal strokes. Compare to the true font, and you'll see the bold style is a more consistent stroke thickness.

The Worst Solution

The way most font-face services provide CSS, each font is defined as an individual font-face declaration. If you write a new set of styles applying these extra font-face declarations to nested bold and italic elements, then the browser still applies faux styles, resulting in double-bold and double-italic text.

The Wrong Solution

To overcome that problem, you add font-weight:normal and font-style:normal rules to your styles to avoid the double-styles, which works, but then if the font fails to load, the backup font styles have no bold or italic.

The Right Solution

The solution is to change the font-face declarations to add styles to the same font-family, which tells the browser to use that font for nested bold and italic styles automatically, and then only rule you apply is the font-family. This solution results in true italic and bold styles, and if the font fails to load, the fallback font still gets proper bold and italic styles.
